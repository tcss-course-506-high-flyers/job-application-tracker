About — Job Application Tracker

A full-stack web application for tracking job applications, managing progress, and staying organized during the job search process.

Team — High Flyers

  • Boma — client-side
  • Aden — database and security
  • Darrell — server-side

Project type

Option 3 from the Week 4 lecture: a Job Application Tracker. Users log and manage their job applications, the app auto-pulls company context from an external API, tracks application stages, and supports follow-up reminders.

Target user

Active job seekers applying to multiple companies at once — students, recent graduates, or professionals transitioning careers. They need a centralized, low-friction way to track applications, follow up on opportunities, and see their pipeline at a glance.

MVP features

  • GitHub OAuth login
  • Add a job application (company, role, date applied, notes)
  • Auto-fetch basic company info via API and cache per company
  • Track application stage (Applied → Interview → Offer → Rejected / Accepted)
  • Move applications between stages with a simple UI
  • Follow-up reminder flagging (e.g. "no update in 7 days")

Endpoints planned

  • GET / — landing page / dashboard
  • GET /login — GitHub OAuth start
  • GET /auth/callback — OAuth callback handler
  • POST /logout — clear session
  • GET /applications — list user's applications
  • POST /applications — create a new application
  • PATCH /applications/<id> — update stage / notes
  • GET /companies/<domain> — fetch (cached) company info

Web-based data source

Primary: Clearbit Company API — returns company logo, description, and industry data by domain. Free-tier limits apply; we minimize calls by caching results per company.

Backup: API Ninjas Company Lookup — used if Clearbit limits become restrictive. Manual entry is the final fallback.

Schema sketch

Three core tables:

  • users — id (PK), github_id, username, email, created_at
  • applications — id (PK), user_id (FK → users), company_domain (FK → companies), role, date_applied, stage, notes, updated_at
  • companies — domain (PK), name, logo_url, description, industry, fetched_at

Why this project

The Job Application Tracker offers a strong balance between technical challenge and real-world usefulness — genuinely an application all of us might use. It exercises OAuth authentication, API integration with fallback strategies, relational data modeling, and interactive UI design. The stage-tracking workflow ensures meaningful client-side interactivity rather than static pages, and aligns well with the skills we want to build this quarter.
